
# Пояснительная записка: WinMaintain

Дата: 2025-12-28

## Зачем это всё
Проблема: набор скриптов (PowerShell/Python/батники) работает, но на практике:
- версии расползаются,
- на чужих ПК нет Python,
- запуск “непрезентабельный”,
- логи и результаты трудно собрать и сравнить.

Цель: превратить это в продукт с понятным входом и воспроизводимыми результатами.

## Почему 2 артефакта (1–2 EXE)
### 1) .NET Launcher (главный)
- один красивый EXE, UI-кнопки, предсказуемые пути,
- внутри `payload.zip`, который распаковывается в `%LOCALAPPDATA%\WinMaintain\toolkit`,
- не зависит от Python и отлично подходит для “дать знакомому/клиенту”.

### 2) Python Scanner (опционально)
- даёт глубокий анализ диска, отчёты, CSV/JSON,
- сборка в EXE через PyInstaller onedir (Python на целевом ПК не нужен),
- подключается как “вторая ступень” — когда нужно точечно разобраться, где теряется место.

## Почему payload.zip
- обновлять toolkit можно отдельно от лаунчера,
- можно хранить батники/ps1/конфиги в одном контейнере,
- распаковка в LocalAppData даёт стабильный путь и не засоряет Downloads.

## Что реально освобождает место
После первичных чисток системные места часто дают мало:
- Windows Temp / Update cache / Recycle Bin становятся “почти пустыми”.

Чаще всего крупные выигрышные зоны:
- AppData (браузеры, UWP Packages, IDE кэши),
- OneDrive (локальные копии),
- JetBrains/VSCode/Cursor/Windsurf кэши,
- офлайн-кэши приложений (музыка и т.п.).

## Принципы “безопасной очистки”
- режим dry-run по умолчанию,
- best-effort: не ломать, а логировать и пропускать,
- не трогать документы/фото/проекты пользователя.
